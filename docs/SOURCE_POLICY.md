# ORIS Source Policy v1

## 1. 目标
统一 ORIS 对“数据来源、证据强弱、链接要求、冲突处理”的执行标准。

---

## 2. 来源优先级

### P1：官方一手来源
优先级最高。
示例：
- 政府机构官网
- 交易所公告
- 公司财报 / 年报 / 季报 / 临时公告
- 官方 API
- 官方白皮书 / 官方文档

### P2：权威二手来源
可作为重要补充。
示例：
- 券商研报
- 权威财经媒体
- 行业研究机构

### P3：普通二手来源
只能补充，不可单独支撑核心结论。

---

## 3. 链接要求
每条核心出处尽量包含：
- 来源机构
- 页面标题
- 发布时间 / 数据时间
- 可访问链接

禁止写法：
- “来源：公开资料”
- “来源：网络”
- “来源：某媒体报道”

必须具体到可核验对象。

---

## 4. 冲突处理
当多个来源冲突时：
1. 优先官方一手来源
2. 若都为官方但口径不同，明确说明统计口径差异
3. 若无法确认真实口径，输出区间判断并标记不确定性

---

## 5. 时效处理

### 强时效主题
以下主题默认高时效：
- 股价
- 财报
- 利率
- 销量 / 交付
- 政策
- 法规
- 宏观数据
- 产品价格
- 排名榜单
- 模型价格 / 能力 / API 状态

这些主题必须优先使用最新来源。

### 弱时效主题
- 历史背景
- 基础概念
- 长期理论
- 稳定定义

---

## 6. 最低出处数量要求

### 单事实问题
至少 1 个可靠来源

### 分析问题
至少 2 个来源，且尽量包含 1 个一手来源

### 高风险分析
至少 3 个来源，且必须优先包含一手来源

---

## 7. 无法提供链接时的处理
若链接无法稳定提供，至少给出：
- 来源机构
- 文档标题
- 发布时间
- 文档类型
- 检索关键词

但这属于降级交付，不是理想状态。
