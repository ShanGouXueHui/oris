# ORIS 来源治理规则

## 核心原则
任何会影响结论可信度的数据，都必须回答两个问题：
1. 数据是什么？
2. 数据出处是什么？

## 最低要求
每条关键数据至少写明：
- 来源名称
- 一个明确定位信息

定位信息优先级：
1. 直接链接
2. 官方报告/公告标题
3. 官方接口名称
4. 日期或时间戳

## 合格示例
- 数据：2026年Q1营收同比增长12.4%  
  出处：公司2026年一季报，2026-04-30，Investor Relations 页面

- 数据：近30日交付量为 18,420  
  出处：公司月度交付公告，官网新闻页，链接见原文

## 不合格示例
- “公开资料显示，公司增长较快”
- “据媒体报道，销量不错”
- “有数据显示，行业景气度提升”

## 报告要求
正式报告默认输出到：
- Word：面向用户阅读与下载
- Excel：评分、证据、明细底表
