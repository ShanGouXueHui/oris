# AGENTS.md

## 当前多 Agent 策略
ORIS 一期默认单主 Agent 执行，暂不开放多 Agent 并行写仓库。

## 原则
- 单任务单主控
- 写入动作集中
- 审计链路统一
- 未来再扩展 code-specialist / research-specialist / ops-specialist

## 强制修改前质量门禁
任何 Agent、Codex 或人工准备修改已存在的工程文件前，必须先执行仓库质量扫描，并检查目标文件是否存在以下问题：

- 同一作用域重复定义；
- JSON 重复 key；
- 明文密码、Token、密钥或其他凭据；
- 环境路径、端口、域名、任务 ID、项目名或 commit SHA 硬编码；
- 文件职责过多或超过规模阈值。

不得跳过扫描后直接修改。若目标文件已有问题，本次改动必须至少不增加问题，并优先同步修复同一职责范围内的问题。

禁止把不同文件、不同进程或不同作用域中的同名局部变量误判为重复定义。运行态会话、队列、执行 evidence 和时间戳快照不是工程源码，不得进行机械重写。

大文件新增功能前必须先拆分职责；新增代码不得继续堆入已超过阈值的单体文件。配置、规则、环境信息和凭据引用必须分别保持单一权威来源。

详细规则见：`docs/DEV_EMPLOYEE_PRE_EDIT_QUALITY_GATE_ADDENDUM_2026-06-19.md`。
